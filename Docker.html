Docker – Phase d’entraînement

1. Contexte et objectifs

Lors de mon stage, j’ai découvert que Docker était un outil essentiel pour déployer et isoler les composants d’une application. Avant d’attaquer le projet principal, mon maître de stage m’a demandé de m’entraîner : il fallait maîtriser la création d’images, le lancement de conteneurs et leur gestion. L’objectif était de prendre confiance avec ce nouveau workflow afin de gagner en autonomie pour la suite du projet.

2. Approche et apprentissage

Pour progresser, j’ai répété plusieurs fois les mêmes commandes Docker jusqu’à ce qu’elles deviennent automatiques. J’ai lu la documentation officielle :

  • hub.docker.com
  • What is a container?
  • docker image ls

Ces lectures m’ont aidé à comprendre la philosophie des conteneurs et à explorer les options de la CLI.

3. Exemples de manipulations

Voici quelques-unes des commandes que j’ai exécutées :

  • docker build -t mon-image . : construire une image à partir d’un Dockerfile.
  • docker images : lister les images locales.
  • docker run -d --name mon-conteneur -p 3000:3000 mon-image : lancer un conteneur en arrière-plan.
  • docker ps : afficher les conteneurs actifs.
  • docker logs -f mon-conteneur : suivre les logs en temps réel.
  • docker stop mon-conteneur && docker rm mon-conteneur : arrêter et supprimer un conteneur.
  • docker rmi mon-image : supprimer une image inutilisée.

En répétant ces opérations, j’ai progressivement réduit le temps de manipulation et intégré ces bons réflexes dans mon quotidien de développement.

4. Technologies utilisées

Outils : Docker Engine, Docker CLI
Images : Node.js, Python, PostgreSQL (officielles)
Environnements : Linux, VS Code

5. Conclusion

Cette phase d’entraînement Docker m’a permis de comprendre les fondamentaux des conteneurs, de m’approprier la CLI et de gagner en confiance avant de basculer sur le projet réel. Je suis désormais capable de construire, déployer et nettoyer mes environnements en quelques commandes, ce qui accélère considérablement le développement et les tests. Surtout, au lieu d’utiliser les Dockerfile et de tester chaque conteneur un par un, on peut utiliser Docker Compose, ce fichier YAML qui regroupe tous les services et qu’il suffit de lancer en une seule commande.
